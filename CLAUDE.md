# ZoneCheck — Claude Code Context

## What this is
ZoneCheck is a timezone overlap tool for distributed teams, a commercial
micro-SaaS under Wolvryn Technology Systems, live at https://zonecheck.wolvryn.tech
No database, no auth, no service worker — intentionally simple and online-only.

## FORGE skills are authoritative
This repo carries the Wolvryn FORGE skills under `.claude/skills/`:
`wolvryn-forge` (core), `-code-standards`, `-security`, `-testing`, `-deploy`.
They define HOW we work (fix philosophy, separation of duties, code/security/
testing standards, commit format, versioning). When this file and a skill

project-specific facts and constraints — it does not restate skill content.

Separation of duties (see `wolvryn-forge`): Claude Code handles implementation
only. Architecture, ADRs, documentation, and commit messages are produced in

## Stack (verified from package.json)
- Next.js 14.2.35 (App Router), React 18, TypeScript 5
- Tailwind CSS 3.4 for all styling (config: tailwind.config.ts)
- date-fns-tz 3.2 for all timezone math
- @anthropic-ai/sdk 0.82, model claude-haiku-4-5-20251001, for AI paste detection
- Vercel hosting (Hobby), auto-deploy on push to main
- @vercel/analytics + @vercel/speed-insights (in app/layout.tsx)

## Hard constraints — do not violate
- NO service worker. App is intentionally online-only.
- NO offline caching of any kind — stale timezone data is worse than none.
- NO database — all state lives in React state and URL query params.
- NO auth in MVP.
- TypeScript strict — no `any`, type guards over `as` assertions (see code-standards skill).
- Tailwind only — no CSS modules, no styled-components.

## File conventions
- Timezone logic: lib/timezone.ts (all of it)
- API routes: app/api/
- The detect route requires `export const runtime = 'nodejs'`
- Secrets via `process.env.ANTHROPIC_API_KEY` — never hardcode, never `NEXT_PUBLIC_*`

## AI detect endpoint
- Route: app/api/detect/route.ts
- Model: claude-haiku-4-5-20251001, max_tokens 150
- Rate limit: 20 requests per IP per 60s, FIXED window, in-memory Map (per
  serverless instance — soft limit, not a global guarantee)
- Returns: { name, timezone, confidence: 'high' | 'low' }
- NOTE: current implementation has open security findings (pending hardening
  pass): no input length cap, raw error messages returned to client, LLM output
  returned unvalidated, user input not delimited in the prompt. See the
  security skill for the standards these must meet.
